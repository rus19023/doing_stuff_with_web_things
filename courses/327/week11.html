How does XQuery interface with MarkLogic document-store database?

MarkLogic extends XQuery with features such as function mapping, serialization options, transaction separator of semi-colon and many others.​ In MarkLogic, since the document-store model is a tree-type structure, a collection of documents (trees) is called a forest, which is the equivalent of traditional database tables. A collection of forests is similar to a traditional database.

MarkLogic provides a web-app GUI tool called Query Console to manage queries. It can be accessed through localhost:8000/qconsole in a web browser, where queries can be built from XQuery, JavaScript, SPARQL and SQL languages. Why is XQuery better than other options with a MarkLogic document-store database?

  • XQuery is supported by all major databases, making it useful for the data ingestion process from many sources.
  • XQuery is case-sensitive, and strings can be enclosed in single or double quotes, so it can do more flexible and specific searches​.
  • MarkLogic has a "universal" index set that is optimized for speed and created when data is ingested, for querying XML elements and JSON properties. This allows XQuery and XPath to be utilized to their full potential.​ The index "sees" structure (schema) the same way it "sees" content.

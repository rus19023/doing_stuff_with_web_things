What is a schemaless collection in MongoDB?

A schema is a well-defined, structured description of each part of a traditional database. This requires rigid formatting to maintain data consistency.​ ​ Schema-less means the schema/model is not required before writing data, but is needed when the data is read, so the data can be translated into what is needed in the warehouse.​ ​ MongoDB does a more natural mapping of data with its JSON-style BSON (binary JSON) documents, so it does not need such structure or definition beforehand. This is what allows for more outward scalability and flexibility in adding fields and nesting levels for the data store.​

Schema-on-write vs. Schema-on-read and Schemaless databases

How do you update data in MongoDB? What happens when an individual field or field value in a document changes?

When a field or field value is changed in MongoDB, the whole document must be re-written to include the new data. This means that the whole document and any embedded documents must be held in memory until the update is complete. This could cause a performance problem if the update is large. The good thing is that each document has a maximum size of 16MB. This means that if you have a large update, you can update the document in chunks.

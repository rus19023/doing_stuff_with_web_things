What is ETL?

ETL is an acronym for Extract, Transform, Load. It is the process used to Extract data from one or more sources, Transform it to match the standards needed for the repository, and Load it into said data warehouse repository. A newer take on ETL is ELT, which stands for Extract, Load, Transform. This is a similar process, only the data is loaded first, then transformed. The purpose is to mitigate coding error, migration speed and server scaling issues during the transformation procedure.

Extract

As there may be many differently formatted data sources, extraction can be a difficult task. First, the data must be profiled to determine the data types and the structure of the data. Version control helps monitor for changes in the data between sessions of exportation.

Transform

Extracted data that is considered "dirty", or detracting from the integrity of the data, must now be "cleaned up", removing duplicate, poorly parsed and/or inconsistent data, correcting misspellings. Data changes tracked can be adjusted as needed, then the data is reformatted to make it uniform for reporting and analysis.

Load

Data collected and structured can now be inserted into fact and dimension tables where it is stored in data cubes that can be "sliced and diced" for reports and analysis as needed.

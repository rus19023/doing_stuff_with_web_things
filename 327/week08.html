Database Management / DevOps

What is the impact of using a database like MongoDB?

MongoDB is easy to start using, even for someone who has never used it before. The documentation is complete and easy to understand for those with limited knowledge. It does not need any schema set up before data can be inserted. The model can be built in the code, using a variety of app programming languages, including at the very least Vanilla Javascript, Python and Node.js.

How is the data stored and what types of challenges will administrators need to deal with in managing this environment?

MongoDB uses the document store type of database storage. Each document is like a table row in a traditional database.

Subqueries can be utilized as documents embedded in the parent documents, for example, a comments document can be embedded within a document containing hike listings for a travel guide. These subdocuments are linked by their auto-generated id numbers to the parent document.

What will be the key considerations in transforming and loading data from the MongoDB environment into a relational database (RDBMS)?

To migrate data from MongoDB to a relational database, the schema for the RDBMS will have to be set up before any records are inserted. If any fields are changed after some records are inserted, the RDBMS schema will need to be altered before any new records can be added. This will put and additional workload on the app developers to be sure the data keeps its integrity.
